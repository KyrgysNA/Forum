Тесты
1. Цели тестирования
Целью тестирования является проверка корректной работы ключевого функционала веб-приложения:
- REST API сервера
- логики работы с базой данных
- отображения данных в пользовательском интерфейсе
- обработки сетевых ошибок

2. Используемые инструменты
Backend:
- Jest — фреймворк для тестирования JavaScript
- Supertest — библиотека для тестирования HTTP-запросов Express-приложения

Frontend:
- Vitest — тестовый раннер
- @testing-library/react — библиотека для тестирования React-компонентов

3. Backend-тесты
Backend-тесты располагаются в директории backend/tests и представляют собой интеграционные тесты API.


3.1 Тесты тем форума (topics.test.js)
Проверяется:
- регистрация и авторизация пользователя
- создание новой темы
- сохранение тегов темы

Сценарий теста:
1. Создаётся тестовый пользователь
2. Выполняется логин и сохраняется cookie
3. Отправляется запрос на создание темы
4. Проверяется успешный HTTP-ответ сервера

3.2 Тесты лайков сообщений (likes.test.js)
Проверяется:
- добавление лайка сообщению
- снятие лайка
- корректный подсчёт количества лайков

Используется toggle-механика (один и тот же эндпоинт).
4. Frontend-тесты

Frontend-тесты располагаются в frontend/src/tests и написаны с использованием testing-library/react в соответствии с лекционными материалами.

4.1 Тест рендера списка тем
Проверяется, что список тем корректно отображается после загрузки данных из API.


4.2 Тест отображения тегов
Проверяется, что теги темы отображаются на странице и имеют корректный текст.


4.3 Тест перехода на страницу темы
Проверяется корректная работа маршрута /topics/:id и отображение сообщений темы.


4.4 Тест обработки ошибок API
Проверяется, что при ошибке сетевого запроса пользователю отображается сообщение об ошибке.
5. Запуск тестов

Backend:
npm test

Frontend:
npm test

6. Вывод
Реализованные тесты покрывают основной функционал проекта. Тестирование подтверждает корректность работы серверной логики и пользовательского интерфейса.
